## Лекция 1

### Что такое программа?

Компьютерная программа — 1) комбинация компьютерных инструкций и данных, позволяющая аппаратному обеспечению вычислительной системы выполнять вычисления или функции управления; 2) синтаксическая единица, которая соответствует правилам определённого языка программирования, состоящая из определений и операторов или инструкций, необходимых для определённой функции, задачи или решения проблемы.

Человеческим языком: 

_Программа_ - комбинация инструкций, написанных на языке программирования, понятном компилятору, и преобразуемая им в комбинацию инструкций, понятных исполнителю.

_Компилятор_ - специальная программа переводящая текст, написанный на языке программирования, в машинные коды.

_Исполнителем_ в данном случае является процессор.

Пример для понимания: заказчик составляет техническое задание, которое способен понять инженер-конструктор; конструктор разрабатывает чертеж, понятный фрезеровщику; фрезеровщик на станке изготавливает деталь. В данном случае роль компилятора выполняет инженер-конструктор.

### Компиляция и интерпретация

Есть несколько видов реализации языков программирования, нам понадобится представление о двух из них: компиляция и интерпретация.

Компиляция подразумевает полное преобразование исходного кода (того самого текста на языке программирования) в машинный код. Плюсы: быстродействие, эффективность, возможность автоматической оптимизации.

Интерпретация подразумевает пошаговое выполнение инструкций исходного кода непосредственно во время вызова. Плюсы: гибкость использования языка.

Разделение это, если что, достаточное условное. Но нам важно знать, что С++ компилируемый язык. Для выполнения программы она должна быть написана вся целиком и компилироваться без ошибок.

Популярным представителем интерпретируемого языка является Python. Программирование можно прекрасно учить и на нем. Так было бы даже легче, но С++ вырабатывает дисциплину, то что нужно на первом курсе.

Вопрос: если компилятор это программа, то на каком языке программирования был написан первый компилятор и как можно было скомпиллировать исходный код для первого компилятора?

### Структура программы

Вспомним понятие функции из математики. 

_y = sin(x)_ - функция синуса.

Здесь _x_ - аргумент функции, то что передается внутрь для вычисления;

_sin_ - название функции, соответствует правилам, по которым обрабатывается _x_;

_y_ - величина которая будет равна посчитанному значению.

В С++ все описывается похожим образом и имеет те же названия: аргумент функции, имя функции. Функция вызывается в тот момент, когда необходимо произвести вычисления, заложенные в функцию, для продолжения работы.

Для правильной работы функции аргумент(ы) для ее вызова должны быть известны. После окончания работы функция возвращает (или нет, если не предусмотрено программистом) какое-то значение.

Функция может выполнять какие-то действия, не обязательно математические. Функция может работать и без аргументов, если так задумано.

Если фам в работе понадобиласть какая-то функция (например математическая), то совсем необязательно ее писать самому. Скорее всего она уже написана до вас, и написана хорошо. Функции схожие по тематике объединяют в специальные сборники - библиотеки. Чтобы не пришлось изобретать велосипед, достаточно подключить соответствующую библиотеку.

Как оформляются функции в C++ (максимально упрощенный вариант без подробностей)

```
<ТИП ДАННЫХ ВОЗВРАЩАЕМОГО ЗНАЧЕНИЯ> <ИМЯ ФУНКЦИИ>(<ТИП ДАННЫХ АРГУМЕНТА><ВНУТРЕННЕЕ ИМЯ АРГУМЕНТА>, ...){
  <ТЕЛО ФУНКЦИИ>
  <ВОЗВРАТ ЗНАЧЕНИЯ>
}
```
`<ТИП ДАННЫХ ВОЗВРАЩАЕМОГО ЗНАЧЕНИЯ>` и `<ТИП ДАННЫХ АРГУМЕНТА>` - определяет, какого рода данные мы используем, мы об этом поговорим позже

`<ИМЯ ФУНКЦИИ>` - то название под которым мы сможем вызвать функцию в программе или даже в другой функции

`<ВНУТРЕННЕЕ ИМЯ АРГУМЕНТА>` - имя, под которым аргумент применяется внутри функции

`<ТЕЛО ФУНКЦИИ>` - тут перечисляются все инструкции/команды, которые функция должна выполнить во время своей работы, например для вычисления конечного значения

`<ВОЗВРАТ ЗНАЧЕНИЯ>` - процедура явного возврата функцией значения в место вызова

__Пример:__

```
int sum_of_xyz(int a, int b, int c){
  int result = a + b + c;
  return result;
}
```

__Пример:__

```
int dummy(){
  return 0;
}
```

При этом вызов таких написанных нами функций будет выглядеть очень похоже на математическую запись:


```
int y;
y = sum_of_xyz(1, 2, 14);
dummy();
```

Теперь вы представляете как выглядит упрощенное описание функции в C++. Зачем это нужно, ведь еще ничего не понятно? Дело в том, что программа на языке C++ это тоже функция. Главная функция, которая внутри себя вызывает другие функции. И оформляется она точно так же как и остальные функции.


```
int main()
{
  return 0;
}
```

Основной особенностью такой _главной функции_ является ее имя, оно должно быть __main__. У главной функции могут быть аргументы, их может не быть (как в примере). Это определяется способом запуска вашей программы, особенностями среды разработки (той программы, в которой вы пишете свои программы).

Когда вам понадобится подключить библиотеку с необходимыми функциями, а вам точно понадобится, нужно будет использовать следующую запись.

```
#include <имя библиотеки>
```

__Пример:__

```
#include <iostream>
```

Как именно называется нужная вам библиотека можно узнать в сети.

Иногда появляется необходимость написать прямо в исходном коде текст, который не должен рассматриваться компилятором, как инструкция. Например, текст комментария на русском языке. Для этого существует два способа. Закомментировать одну строчку (ее часть) или закомментировать целый блок текста.

Комментриование строки или ее части можно поставив `\\`. После этих символов и до конца строки все написанное будет игнорироваться компилятором и не считаться программой.
Комментирование блока текста достигается использованием `\*` и `*\` для указания начала и конца комментируемой области.

__Пример__

```
\\Отдельная строка с комментарием
a = b; \\Этот способ позволяет оставить комментарий напротив нужного места без выделения лишней строки
\* Тут можно закомментировать целый блок из нескольких строчек
a = b + 1;
b = a + 1; *\
```

Используя комментарии можно "выключать" часть исходного кода, которы вы не хотите стирать. Никогда не стирайте работающий код, лучше его комментировать. Когда программа будет полностью работоспособна, можно будет навести красоту.

Еще одно важная вещь, которую вы уже могли заметить. Каждая инструкция в C++ заканчивается точкой с запятой ";".
